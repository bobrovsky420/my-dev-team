## Code Style

- Follow PEP 8 guidelines for code formatting.
- Use single quotes for literals, like dict keys, path and file names, etc.
- Use double quotes for meaningfull strings, like human readable messages including log messages.
- Do not write docstrings for modules.
- Write docstring for classes and methods only if they are absolutely necessary and the class or method name itself is not self-explanatory.
- Write comments only if they are absolutely necessary and the code itself is not self-explanatory.
